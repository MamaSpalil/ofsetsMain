# MuOffsetLogger

Консольное EXE-приложение для анализа и логирования офсетов, переменных и функций main.exe MU Online.

## Описание

MuOffsetLogger.exe — автономное консольное приложение. Логика работы:
1. Помещаем `MuOffsetLogger.exe` в папку с игровым клиентом (рядом с `main.exe`)
2. Запускаем `MuOffsetLogger.exe` — открывается консольное окно
3. Программа автоматически ищет `main.exe`: если процесс уже запущен — подключается к нему, иначе предлагает запустить
4. Анализ PE-структуры, автоматическая классификация всех офсетов из файла main.exe
5. **Окно остаётся открытым** — запускается режим мониторинга main.exe
6. Отслеживаются все действия в окне игры: фокус, перемещение, свёрнутость, заголовок
7. Мониторинг продолжается до закрытия main.exe или нажатия Q/ESC
8. Логи отображаются в консоли и сохраняются в `MuOffsetLog.txt`

### Этапы анализа

1. **PE Structure Analysis** — разбор заголовков PE, секций, таблицы импорта (IAT)
2. **Known Offsets Database** — вывод известных офсетов с комментариями и описаниями (включая игровые данные)
3. **Function Scanning** — сканирование .text секции: поиск функций по прологам (55 8B EC) и CALL-целям
4. **String Reference Scanning** — поиск строковых ссылок в секции данных
5. **Auto-Classification** — автоматическая классификация функций по вызовам импортов и строковым ссылкам (версионно-независимо)
6. **Process Monitoring** — отслеживание окна main.exe в реальном времени
7. **Game Monitoring** — отслеживание ВСЕХ игровых действий через чтение памяти процесса

### Режим мониторинга

После анализа файла программа переходит в режим мониторинга процесса main.exe:

#### Мониторинг окна (ProcessMonitor)
- **Окно игры**: создание, уничтожение, пересоздание
- **Заголовок окна**: изменения заголовка (индикация состояния игры)
- **Фокус**: получение/потеря фокуса (передний план / фон)
- **Позиция и размер**: перемещение и изменение размера окна
- **Состояние**: свёрнуто/развёрнуто, видимо/скрыто
- **Процесс**: запуск, завершение, код выхода

#### Мониторинг игровых действий (GameMonitor)
- **Выбор сервера**: группа серверов, индекс, имя сервера
- **Ввод логина/пароля**: поля ввода, длина, результат авторизации
- **Выбор персонажа**: количество персонажей, выбранный слот
- **Данные персонажа**: имя, уровень, класс, HP/MP, позиция, карта, опыт
- **Инвентарь**: количество предметов, изменения
- **Ближайшие игроки**: имена, HP, позиция, появление/исчезновение
- **Ближайшие монстры**: имена, HP, позиция, появление/смерть
- **Чат**: последние сообщения чата
- **Бой/Урон**: нанесённый/полученный урон (игрокам и монстрам)
- **Убийство монстров**: счётчик убийств
- **Уровень**: повышение уровня
- **Телепортация**: смена карты, координаты назначения
- **Клавиатура**: нажатия всех клавиш (VK-коды)
- **Мышь**: позиция курсора, нажатия кнопок (ЛКМ/ПКМ/СКМ)

## Требования для сборки

- **Visual Studio 2010** (Platform Toolset v100)
- **Windows SDK** (встроенный в VS 2010)
- **ОС**: Windows 10 x86 или x64

## Сборка проекта

### Через Visual Studio 2010

1. Откройте `MuOffsetLogger/MuOffsetLogger.sln` в Visual Studio 2010
2. Выберите конфигурацию:
   - `Debug|Win32` или `Release|Win32` — для 32-битного EXE
   - `Debug|x64` или `Release|x64` — для 64-битного EXE
3. Нажмите `Build → Build Solution` (Ctrl+Shift+B)
4. Готовый файл `MuOffsetLogger.exe` будет в папке `Debug/` или `Release/`

### Через командную строку (Developer Command Prompt)

```bat
cd MuOffsetLogger
msbuild MuOffsetLogger.sln /p:Configuration=Release /p:Platform=Win32
```

> **Важно:** main.exe MU Online — 32-битный PE32 файл. Рекомендуется собирать как **Win32** (x86) для совместимости.

## Использование

1. Соберите проект (конфигурация `Release|Win32`)
2. Скопируйте `MuOffsetLogger.exe` в папку с игровым клиентом MU Online (рядом с `main.exe`)
3. Запустите `MuOffsetLogger.exe`
4. Программа автоматически найдёт main.exe:
   - Если main.exe уже запущен — подключится к нему для мониторинга
   - Если main.exe не запущен — предложит запустить:
     - Введите `1` — main.exe запустится, начнётся анализ и мониторинг
     - Введите `0` — только анализ файла без запуска
5. **Окно программы остаётся открытым** — идёт мониторинг main.exe в реальном времени
6. Все события (фокус, перемещение окна, свёрнутость) логируются в консоль и файл
7. Для остановки мониторинга нажмите **Q** или **ESC**
8. После остановки нажмите любую клавишу для закрытия программы
9. Результаты сохранены в `MuOffsetLog.txt`

## Формат вывода

Каждый офсет выводится в формате:
```
VA  (File: FileOffset)  [Type]  Name -- Description
```

Где:
- **VA** — виртуальный адрес в памяти
- **FileOffset** — смещение в файле main.exe
- **Type** — тип: FUNC (функция), VAR (переменная), STR (строка), IAT (импорт), FLOAT (константа), CRT (Runtime), VTBL (виртуальная таблица)
- **Name** — имя функции или переменной
- **Description** — описание назначения

## Данные main.exe

- **Тип**: PE32 (GUI) Intel 80386, Windows
- **Размер**: 4,316,672 байт (0x0041DC3D)
- **ImageBase**: 0x00400000
- **Точка входа**: 0x0917C200 (секция .LibHook, защита ASProtect)
- **Секция .text**: VA 0x00401000, Size 0x003B2000 — основной код
- **Секция .data**: VA 0x007B3000, Size 0x0001C000 — глобальные данные
